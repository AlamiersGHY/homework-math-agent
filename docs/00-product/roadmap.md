# 产品路线图

本文档定义 Math Agent 的阶段性交付路线。路线图服务于短周期 AI Coding：每个阶段都应形成可验证的产物，而不是只完成抽象设计。

## Phase 0：SDD 基座与 Eval 基座

目标：让后续 AI Coding 有稳定上下文、文档入口和行为验收锚点。

交付物：

- `AGENTS.md` 全局工作协议。
- `docs/` SDD 文档骨架。
- `evals/` 行为评估入口。
- 产品范围、架构方向、测试策略的最小可执行文档。

完成标准：

- 新 Agent 能通过 `AGENTS.md`、`docs/INDEX.md`、`docs/04-logs/active.md` 找到当前任务和产品范围。
- MVP 范围不再只存在于对话记忆中。

## Phase 1：最小对话 Agent 与回答模式

目标：跑通文本输入到数学回答的最小闭环。

核心能力：

- 文本问题输入。
- 题型分类：概念类、计算类、证明类、可视化类。
- 回答模式：直接解答、分步引导、仅提示。
- LaTeX 风格数学输出。
- 基础多轮上下文或提示轮次状态。

完成标准：

- 用户能完成一次可用数学问答。
- 用户能切换“直接答案”和“引导提示”。
- Agent 行为有初始 eval cases 覆盖。

## Phase 2：OCR 输入链路

目标：降低数学题输入门槛，让图片题目能进入对话流程。

核心能力：

- 图片上传入口。
- OCR provider 抽象。
- OCR 识别结果展示。
- 用户可编辑确认识别结果。
- 确认后的文本进入 Phase 1 对话链路。

完成标准：

- 上传图片后能得到可编辑文本。
- OCR 失败时有明确降级提示。
- OCR provider 可以后续替换，不写死在业务流程中。

## Phase 3：3D/函数可视化链路

目标：形成项目特色亮点，用图形增强函数、积分和空间曲面的理解。

核心能力：

- Agent 判断题目是否适合可视化。
- 生成基础 plot spec。
- 前端渲染 2D 函数或 `z = f(x, y)` 曲面。
- 图形支持基础交互查看。
- 回答中配合解释图形代表的数学意义。

完成标准：

- 至少能稳定演示一类 3D 曲面图。
- 至少能稳定演示一类 2D 函数或区域图。
- 可视化触发行为有 eval cases 覆盖。

## Phase 4：Agentic RAG Prototype

目标：把本地 MVP 学习工作台升级为 chat-first 智能课程助手原型。用户自然提问，系统自动判断是否需要课程资料检索、引用、可视化、澄清、偏好或记忆。

### Phase 4.1：Agent Planner Skeleton

核心能力：

- 后端 planner service。
- 结构化 planner 输出，而不是自由文本。
- Planner metadata 进入 `POST /chat/stream` SSE。
- 覆盖检索、绘图、澄清、回答模式、记忆动作等决策字段。

完成标准：

- 三重积分、空间区域或曲面问题能触发 `needs_plot=true`。
- 概念解释类问题能触发 `needs_retrieval=true`。
- 证明题倾向 `guided` 或 `hint`。
- 普通计算和离题输入不强制触发工具。
- Planner 行为有 deterministic tests/evals 覆盖。

### Phase 4.2：PDF / 课程材料检索 v1

核心能力：

- PDF/课程材料上传。
- 文本提取和 chunking。
- 文档、chunk、页码、章节等本地元数据。
- 简单关键词、embedding 或 hybrid retrieval。
- 检索失败不阻塞普通回答。

完成标准：

- 上传材料能被查询。
- 检索结果包含真实文件名、页码和文本片段。
- 空检索不编造来源。

### Phase 4.3：Citation-Aware Answering

核心能力：

- 检索结果进入回答 prompt。
- 前端展示来源/citation 区域。
- 回答区分课程材料证据和通用模型推理。

完成标准：

- 概念问题能引用上传材料。
- 缺少材料证据时明确说明，而不是虚构页码或文件。
- citation safety 有 eval 覆盖。

### Phase 4.4：自动工具执行

核心能力：

- Planner 决策驱动 retrieval、plot、clarification 等工具。
- 支持自动 plot 附件和 citation metadata。
- 用户无需手动选择每个工具。

完成标准：

- 支持的空间区域/曲面问题能自动附图。
- 课程概念问题能自动检索。
- 信息不足时能先提出澄清。

### Phase 4.5：Preferences + Lightweight Memory

核心能力：

- 本地偏好设置。
- 自定义回答风格和自动工具开关。
- 有边界的学习记忆和弱点记录。

完成标准：

- 默认风格和自动工具偏好可保存并影响回答。
- 记忆注入有上限，不污染无关回答。

## Phase 5：课程助手 Polish / 部署反馈

目标：把 Agentic RAG Prototype 变成可访问、可演示、可复盘的课程助手原型，并在不拖慢核心闭环的前提下提高概念解释和练习推荐的专业性。

核心能力：

- 前后端联调和基础部署路径。
- 基础错误处理、加载状态和演示路径。
- 小规模真实用户试用。
- 少量例题、复习建议或相似题推荐。
- 记录产品反馈和 AI Coding / SDD 实践总结。

完成标准：

- 有线上可访问版本。
- 有演示路径和基础使用说明。
- 记录关键问题、技术债和后续改进方向。

## 里程碑取舍原则

- 每个阶段都应保持可运行或可验证。
- OCR 和可视化是核心体验，不应无限后置。
- RAG、题库和个人知识库是增强能力，不应阻塞核心闭环。
- 如果开发周期紧张，优先保留“输入 -> 回答模式 -> 可视化/OCR 亮点 -> 可部署演示”的主线。
